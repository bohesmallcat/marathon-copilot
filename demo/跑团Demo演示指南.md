# Marathon Copilot — 跑团 Demo 演示指南

> 面向马拉松爱好者的项目介绍 & 现场演示脚本

---

## 一、开场：一个真实的故事（2 分钟）

> **建议话术：**
>
> "大家有没有遇到过这种情况——赛前纠结半天'我到底该跑多快'，结果比赛一出发就跟着大部队冲了，后半程崩得怀疑人生？
>
> 或者赛前看了天气预报说 15 度、多云，觉得是 PB 天气，结果到了现场湿度 90%，跑到半程心率就飘了？
>
> 我自己就经历过。有一场春季半马，赛前目标 1:32，结果跑了 1:35——多了整整 3 分钟。后来复盘发现，赛道有一段隧道湿度 90% 以上、CO₂ 浓度高、体感温度比外面高 3-5 度，那段路心率直接炸了。
>
> 所以我做了这个工具，**用数据帮你做长期训练规划、赛前决策、盯天气变化、赛后找原因**。今天给大家演示一下。"

## 三、分场景 Demo（20-25 分钟）

### 场景 1：长期训练规划 — "离比赛还有 4 个月，我该怎么练？"（4 分钟）

**痛点共鸣：**
- "买了训练计划但跟不上，或者太简单没挑战"
- "不知道每周跑量应该增加多少，怕受伤"
- "女生跑者，生理期前后状态差异大，不知道怎么调整"

**展示内容：** 打开长期训练规划方案 PDF

| 展示要点 | 说什么 |
|----------|--------|
| VDOT 递进路径 | "系统根据你当前 VDOT 和目标成绩，规划每 16 周提升 0.5-1.5 点的递进路径" |
| 分期训练阶段 | "恢复期 → 有氧基础期 → 有氧发展期 → 专项期 → 巅峰期 → 减量期——每个阶段的训练重点和跑量都不同" |
| 跑量递进规则 | "严格遵守 10% 周增量上限，每 4 周有一个减载恢复周。体重大的跑者自动收紧到 8%" |
| 里程碑测试 | "中间安排 5km、10km、半马测试赛，验证进步是否达预期，不达标自动调整方案" |
| 多因子达成概率 | "不是拍脑袋说你能不能破 PB，而是用 VDOT 缺口、训练年限、跑量、伤病、耦合比等多因子精算概率" |
| 女性生理周期 | "自动预测比赛日处于什么周期相位，给出对应的训练强度调整和赛日策略" |

**互动点：** "在座如果告诉我你的目标比赛和当前 PB，我可以告诉你大概需要几个训练周期。"

---

### 场景 2：每周训练计划 — "这周每天练什么？"（3 分钟）

**痛点共鸣：**
- "周三天气预报下暴雨，我的间歇跑怎么办？"
- "力量训练不知道做什么，网上方案跟跑步没关系"

**展示内容：** 打开本周训练计划 PDF

| 展示要点 | 说什么 |
|----------|--------|
| 天气自适应排课 | "自动获取 7 天天气预报，质量课排在好天气日，恶劣天气日安排休息或室内交叉训练" |
| 每日训练处方 | "精确到配速、心率上限、RPE、距离——不是笼统说'跑 10 公里'，而是告诉你具体怎么跑" |
| 恢复监控决策树 | "每天检查晨脉、DOMS、疼痛量表、睡眠，系统自动判断今天该正常练还是减量" |
| 力量训练课表 | "随训练阶段变化：基础期侧重关节稳定，发展期侧重跑步经济性，赛前维持即可" |
| 饮食方案 | "碳水/蛋白质/脂肪按体重和训练阶段计算，训练日和恢复日的营养比例不一样" |

**互动点：** "大家有没有因为天气打乱过训练计划？这个功能就是自动帮你重新排课。"

---

### 场景 3：赛前目标设定 — "我该跑多快？"（4 分钟）

**痛点共鸣：**
- "PB 跑过 330，这次想破 320，靠谱吗？"
- "上次比赛前三个月没怎么练，目标要不要保守点？"

**展示内容：** 打开赛前目标设定报告 PDF

| 展示要点 | 说什么 |
|----------|--------|
| VDOT 跑力值计算 | "根据你过去两场全马 PB，系统算出 VDOT——这个数字代表你的真实跑力" |
| 多场比赛数据比对 | "系统会对比你不同赛事的数据，发现哪次的配速策略更优，掉速率更低" |
| A/B/C 三档目标 | "不是只给你一个目标，而是给三档：A 目标冲 PB、B 目标稳完赛、C 目标保底" |
| GPS 距离修正 | "手表显示 42.195 但实际你跑了 42.6km——系统会修正这个偏差，避免你配速算错" |

**互动点：** "在座的如果告诉我你的半马 PB，我可以当场算出你的 VDOT 和全马预测成绩。"

---

### 场景 4：赛前天气追踪 — "天气对我影响多大？"（4 分钟）

**痛点共鸣：**
- "天气预报说 15 度，但到底是有利还是不利？"
- "湿度 80% 意味着我要慢多少？"

**展示内容：** 打开赛前天气日报 PDF

| 展示要点 | 说什么 |
|----------|--------|
| 天气速览面板 | "一眼看清比赛日天气：10°C→15°C，湿度 69%→52%，无降水——绿灯" |
| **环境税** 概念 | "这是核心概念：天气会给你的配速'加税'。今天的环境税是 10 秒/公里——意味着同样的心率，你会比理想天气慢 10 秒" |
| 逐项拆解 | "风 +2 秒、湿度 +0 秒、紫外线 -5 秒……每一项都量化了" |
| 红绿灯判断 | "系统会告诉你：当前天气是执行原计划，还是要启动 B 方案降速" |

**对比展示：** 再打开另一场赛前天气数据 —— "同样是春天的比赛，另一场环境税高达 92 秒/公里。赛道有一段隧道湿度 90%+，这就是为什么那天跑崩了。"

**互动点：** "大家想想自己上次比赛，如果跑前知道环境税是多少，会不会做不同的配速决策？"

---

### 场景 5：每日训练 & 饮食计划 — "赛前一周该怎么练、怎么吃？"（3 分钟）

**痛点共鸣：**
- "赛前一周到底该跑多少？不敢练也不敢不练"
- "糖原超补到底怎么补？吃多少碳水？"

**展示内容：** 打开微信小程序的"每日训练 & 饮食"页面（或展示截图）

| 展示要点 | 说什么 |
|----------|--------|
| 赛前倒计时 | "进去就看到 D-7、D-6……每天该做什么一目了然" |
| 训练计划 tab | "D-7 轻松跑 6km 配速 5:30，D-3 减量到 3km，D-1 赛前激活跑 2km——配速都是根据你的 VDOT 算出来的" |
| 饮食计划 tab | "D-3 开始糖原超补：碳水从每公斤体重 5g 提高到 8g、10g。60kg 的跑者就是 480g→600g 碳水" |
| 恢复建议 tab | "D-2 建议泡沫轴放松 + 8 小时睡眠，D-1 赛前不做任何高强度拉伸" |

**互动点：** "赛前你们一般怎么安排饮食？有没有算过每天吃多少克碳水？"

### 场景 6：比赛执行方案 — "比赛当天看什么？"（3 分钟）

**痛点共鸣：**
- "站在起跑线上，脑子里只记得'目标 330'，但每公里该跑多少完全没数"
- "跑到 30 公里心率飙到 180，是该坚持还是降速？"

**展示内容：** 打开比赛执行方案 PDF

| 展示要点 | 说什么 |
|----------|--------|
| 赛前状态速览 | "一张表：HRV 状态评估、PB 胜率、环境挑战" |
| 逐公里配速表 | "每段该跑多快，从热身到巡航到冲刺，每个阶段的目标配速和心率——**手机锁屏就能看**" |
| 熔断机制 | "如果某段心率超标或配速偏差过大，系统建议你立刻切换到 B 方案。**这不是认输，是科学止损**" |
| 补给时间表 | "每 5km 一个补给点，几点喝水、几点吃胶、吃什么口味——都帮你排好了" |

**互动点：** "有没有人比赛时用过配速手环？这个方案比手环多了心率控制和熔断规则。"

### 场景 7：赛后复盘 — "我为什么崩了？"（5 分钟）

**痛点共鸣：**
- "跑完就看个总成绩和平均配速，不知道到底哪里出了问题"
- "30 公里撞墙，是体能不够还是前面跑太快？"

**展示内容：** 打开赛后复盘报告 PDF

| 展示要点 | 说什么 |
|----------|--------|
| 计划 vs 实际对比 | "逐 5km 分段对比配速、心率、步频——不只是看总成绩" |
| 分段执行力分析 | "哪些段完美执行，哪些段开始崩溃，一目了然" |
| **四维关联分析** | "这是最有价值的部分：步幅 × 步频 × 功率 × 配速——找出掉速的生理原因。比如步幅衰减 10% 是掉速的主因，不是步频——说明是肌肉疲劳导致步幅缩短" |
| **步幅衰减三阶段模型** | "系统自动识别稳定期 → 加速衰减期 → 崩塌期的里程拐点，量化步幅缩短贡献了多少配速损失" |
| **心率-配速脱耦诊断** | "区分'心率漂移型'衰减（湿度/脱水）和'糖原耗竭型'崩盘（配速暴降但心率不升反降），精准定位根因" |
| **偏差分解量化** | "总偏差精确分解到各因子：跑量不足 +X 秒、赛间恢复不充分 +Y 秒、步幅崩塌 +Z 秒……总和必须吻合" |
| 跨赛事对比 | "多场比赛横向对比，找出训练和执行的规律" |

**互动点：** "你们跑完比赛一般怎么复盘？如果手表数据导出来，这个工具可以帮你做这种级别的分析。"

## 四、真实案例对比（2 分钟）

展示同一场景两场比赛的对比，最有说服力：

| | 春季半马 A（未使用 Copilot） | 春季半马 B（使用 Copilot 后） |
|---|---|---|
| 赛前准备 | 凭感觉定配速 | VDOT 算目标 + 天气修正 |
| 天气应对 | 没注意到隧道高湿 | 环境税量化，绿灯执行 |

> "第一场没有数据支撑，根因分析发现是赛道微气候——隧道段湿度 90%+、体感温度高 3-5 度。**如果赛前知道这些，至少可以在隧道段主动降速保心率，而不是被动崩掉。** 第二场基于上次复盘数据，做好了完整的天气追踪和执行方案。"

## 五、怎么开始用？（2 分钟）

### 方式 1：微信小程序（面向所有跑者）

```
打开小程序 → 建档案（填 PB + 体重） → 选比赛 → 自动出方案
```

功能入口：
- 赛前目标设定
- 每日训练 & 饮食
- 赛后复盘
- 历史记录

### 方式 2：找我帮你生成（目前阶段）

> "目前小程序还在迭代中，如果你有即将到来的比赛，可以把你的 PB 数据和赛事信息发给我，我用 CLI 工具帮你生成完整的作战方案和每日简报。"

需要提供的信息：
- 身高、体重
- 历史 PB（最好有最近 2-3 场比赛的分段数据）
- 目标比赛名称、日期、距离
- 手表品牌（方便导出数据格式）
- HRV 数据（如果有的话）
- 女性跑者：生理周期信息（可选，有则更精准）

## 六、Q&A 预设问题

| 可能的问题 | 建议回答 |
|------------|----------|
| "这个和佳明/COROS 自带的训练计划有什么区别？" | "手表给你的是通用训练计划，我们做的是**比赛执行方案**——结合你的真实跑力、当天天气、身体状态，给出个性化的逐公里配速和熔断规则。另外我们还有长期训练周期化规划和深度赛后复盘，这些手表做不到。" |
| "数据准不准？" | "核心算法基于 Jack Daniels 的 VDOT 公式，这是全球最主流的跑力评估体系。环境修正模型参考了运动生理学文献。赛后复盘的四维分析和根因链也有文献支撑。但任何模型都有局限——所以我们给三档目标，而不是只给一个数字。" |
| "我只跑半马，能用吗？" | "完全可以。半马和全马都支持，目标设定、天气追踪、赛后复盘都一样。" |
| "我是初跑者 / 430 水平，对我有用吗？" | "越是初跑者越需要科学配速。430 水平最常见的问题就是前半程跟着人群跑太快、后半程撞墙。一份靠谱的配速表能帮你少走很多弯路。长期训练规划更是初跑者最需要的。" |
| "需要付费吗？" | "目前完全免费，开源项目。" |
| "我的数据安全吗？" | "你的个人数据只在本地处理，不会上传到任何第三方服务器。代码仓库里已经做了完整的脱敏处理。" |
| "女性跑者有什么特别功能？" | "系统会根据你的生理周期信息，自动调整训练强度、营养比例，还会做 ACL 风险预警和 RPE 膨胀修正。比赛日如果碰到黄体期，会自动调低配速预期。" |
